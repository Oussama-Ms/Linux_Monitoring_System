**SecureBox** is a robust, multi-threaded systems programming utility designed to monitor local Linux directories in real-time. Built natively in C, it leverages the Linux `inotify` API to detect file system events (creation, deletion, modification, and renaming) with minimal CPU overhead, making it an efficient tool for identifying suspicious activities and maintaining data integrity.

---

## 🏗️ Architecture

This project strictly adheres to POSIX standards and utilizes a **Multi-Threaded Architecture**:
* **Monitor Thread:** Acts as the producer, utilizing `inotify` to passively listen for kernel-level filesystem events without blocking or polling.
* **Analysis & Logging Threads (Upcoming):** Act as consumers, processing the event queue using mutexes and condition variables to ensure thread safety and prevent race conditions.
* **Graceful Degradation:** Implements signal handling (`SIGINT`, `SIGTERM`) to ensure proper resource deallocation and prevent memory leaks.

## 🚀 Getting Started (Launch Tutorial)

To ensure a perfectly synchronized development environment across all machines, this project utilizes **VS Code Dev Containers**. This guarantees that the compiler (`gcc`), memory checking tools (`valgrind`), and build systems (`make`) are identical for every contributor.

### Prerequisites
1. **Docker Desktop** (Configured with the WSL2 backend if you are on Windows).
2. **Visual Studio Code**.
3. The **Dev Containers** extension installed in VS Code.

### Installation & Execution

**Step 1: Clone the repository**
```bash
git clone [https://github.com/Oussama-Ms/securebox.git](https://github.com/Oussama-Ms/securebox.git)
cd securebox
```
**Step 2: Start the Dev Container**

Open the folder in VS Code.

A prompt will appear in the bottom right corner: "Folder contains a Dev Container configuration file. Reopen to develop in a container." * Click Reopen in Container. (Docker will automatically build the isolated Linux environment).

**Step 3: Build the application**

Open a new terminal inside VS Code (you are now inside the Linux container) and run:

```Bash
make
```
**Step 4: Run the Monitor**

Note: The inotify API requires a native Linux filesystem to detect kernel events. Do not run the monitor on folders mounted from Windows (like your workspace). Use the native /tmp directory for testing.

```Bash
# Create a test directory
mkdir /tmp/test_dir

# Launch SecureBox
./securebox /tmp/test_dir
```
## 🕹️ Usage & Testing

Once SecureBox is running, it will listen for events in the background.
To test the real-time detection, open a second terminal in VS Code and trigger filesystem events in the target directory:

```Bash
cd /tmp/test_dir

# 1. Trigger a CREATE event
touch secret_data.txt

# 2. Trigger a MODIFY event
echo "SecureBox is watching" > secret_data.txt

# 3. Trigger a RENAME event
mv secret_data.txt public_data.txt

# 4. Trigger a DELETE event
rm public_data.txt
```
Switch back to your first terminal to see the live event logs generated by the monitor thread.
